ScalaTest/Scalactic 3.1.0 Release Notes

ScalaTest/Scalactic 3.1.0 (for Scala 2.10, 2.11 and 2.12) includes the enhancements and bug fixes listed below. No source code using ScalaTest/ScalaUtils 3.0 or higher should break, but from any version other than other 3.0.x releases (with which this release is binary compatible), you will likely need to do a clean build to upgrade.

For information on how to include ScalaTest in your project, see the install page. For information on how to use Scalactic in your production code, see its install page.

 

Scalactic

New Features

  • Added NonZero AnyVals.
  • Added NumericString AnyVals.
  • Added Neg and NegZ AnyVals.
  • Added Present, a wrapper class to allow any value to be used with 'for'.
  • Added PosFinite and PosZFinite AnyVals.
  • Added NegFinite and NegZFinite AnyVals.
  • Added NonZeroFiniteFloat and NonZeroFiniteDouble AnyVals.
  • Added FiniteFloat and FiniteDouble AnyVals.
  • Added NonEmpty AnyVals.
  • Added RegexString AnyVals (still mark as private).

Enhancements

  • Use the isValid method technique in all built-in AnyVals. Add ensuringValid methods to all AnyVal companion objects.
  • Use ensuringValid in the macro to avoid the very temporary Option instance it had been creating before because of idealism. Pragmatism has won again in the end!
  • Renamed Percent to PercentageInt.
  • Changed DigitChar to NumericChar.
  • Use ensuringValid instead of from/get in implementations of AnyVals methods.
  • Added Scaladoc to ensuringValid method in Pos* AnyVals.
  • Added isValid and fromOrElse methods to all the Pos* AnyVals types.
  • Added an ensuringValid method that takes an X => X function to each of the PosX and PosZX.
  • Added MinPositiveValue constants to PosDouble and PosZDouble companion objects.
  • Added a sumOf and productOf method to companion object of PosDouble, PosZDouble, PosFloat and PosZFloat.
  • Added plus method to PosFloat and PosZFloat.
  • Added tryingValid, passOrElse, goodOrElse and rightOrElse methods to PosInt.
  • Improved Scaladoc on Prettifier.
  • Changed negation methods in AnyVals to return another anyvals type.

Bug Fixes

  • None

Deprecations

  • Moved org.scalactic.Chain as org.scalactic.anyvals.NonEmptyList, deprecated Chain in org.scalactic.

 

ScalaTest

New Features

  • Added asynchronous support to Inspectors.
  • Added asynchronous support to TableAsserting.
  • Added asynchronous support to WheneverAsserting.
  • Added JSON support to events and StringReporter.
  • Added Ultimately, which is an asynchronous version of Eventually.

Enhancements

  • Enhanced WebBrowser's isInputField method to treat "input element that returns null when getType is called" as a 'text'-type input element
  • Enhanced WebBrowser's value_= method to execute javascript to set value when the driver being used is JavascriptExecutor.
  • Added toDir method to WebBrowser.
  • No longer use .toLowerCase when looking at the value of the environment variable for filling file pathnames; it now looks for the value "yes", not "true".
  • Removed an unnecessary nested synchronized call in SuiteSortingReporter.
  • Made MessageRecorder thread-safe.

Bug Fixes

  • Fixed macro-based assertion that had a problem when used directly with package name without mixin and import of the trait. Before this fix, the following will not work: org.scalatest.Assertions.assert(a1 === a3) The user had to mix in Assertions into their class and use: assert(a1 === a3) After this fix, both of the cases now work.

Deprecations

  • In FeatureSpec, changed 'feature' to 'Feature'. Deprecated 'feature'.
  • In FeatureSpec, changed 'scenario' to 'Scenario'. Deprecated 'scenario'.
  • In FeatureSpec, changed 'scenariosFor' to 'ScenariosFor'. Deprecated 'scenariosFor'.
  • Moved Checkers under org.scalatest.prop to org.scalatest.check package, deprecated Checkers under org.scalatest.prop.
